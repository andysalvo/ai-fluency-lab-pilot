# Pilot Scope (Frozen)

## Problem Focus
How do we build sustained AI fluency inside a student population when the technology and norms are constantly shifting?

## In Scope
1. Notion-first student thread workflow.
2. Local and System modes with explicit boundaries.
3. Output logging with readiness gate and versioning.
4. Research Inbox -> Librarian -> Moderator -> Library workflow.
5. Supabase structured memory and run audit.
6. 48-hour vertical slice demo.
7. One active root problem statement (2-week lock) that all threads reason downstream from.
8. Position Journey:
   - Individual Positions (thread-level)
   - Cohort Position (weekly, approved)
   - visible directional evolution over versions

## Out of Scope
1. Autonomous orchestration.
2. Auto-merge thread behavior.
3. Hidden cross-thread reads.
4. Full production architecture and scaling optimization.
5. Secret values in repository files.
6. Autonomous consensus generation or auto-published cohort decisions.

## Success Orientation
1. Thread continuity across multiple weekly cycles.
2. Explainable, provenance-backed system reasoning.
3. Controlled automation with human approval gates.
4. Downstream continuity from root problem to weekly cohort position.
5. Position evolution quality (hold/shift/strengthen/split) with provenance.
